# Patterns Reference

Reusable UI patterns generated by `/app-pattern <name> [--into <file>]`.

## Catalog

| Pattern | File | Purpose |
|---|---|---|
| `data-table` | `assets/patterns/data-table.tsx` | Sortable table with row actions. Uses `TBL`. |
| `empty-state` | `assets/patterns/empty-state.tsx` | Centered illustration + message + CTA. |
| `loading-skeleton` | `assets/patterns/loading-skeleton.tsx` | Shimmering placeholder blocks. |
| `hero` | `assets/patterns/hero.tsx` | Landing-page banner with headline + CTA. |
| `pricing-grid` | `assets/patterns/pricing-grid.tsx` | Three-tier pricing cards. |
| `notification-toast` | `assets/patterns/notification-toast.tsx` | Dismissible toast with aria-live. |

## Insertion modes

**Standalone** (no `--into`): creates `src/patterns/<Name>.tsx`.

**Inline injection** (`--into=src/pages/Overview.tsx`): the pattern is inserted at the first `{/* @acss-app-builder:insert */}` marker in the target file. If the marker is absent, the command refuses and prints the exact comment to add.

## Component-source adaptation

Each pattern template declares `{{IMPORT_SOURCE:...}}` at the top. On insertion:

1. The active import map is resolved via `detect_component_source.py`.
2. Imports are substituted relative to the target file's depth.
3. SCSS imports are emitted for `source=generated`.

## Accessibility per pattern

- **data-table**: `<caption>` required; row actions are `<button>` with `aria-label` if text-only icon.
- **empty-state**: `role="status"` on the message container so SR announces.
- **loading-skeleton**: `aria-busy="true"` + `aria-label="Loading"` on the parent.
- **hero**: `<h1>` only if the pattern is the page's primary hero; otherwise demoted.
- **pricing-grid**: each card is a `<section>` with `<h3>` heading; "recommended" has `aria-current="true"`.
- **notification-toast**: `role="status"` for non-critical, `role="alert"` for errors; auto-dismiss does not remove focusable dismiss button.

## Styling

Patterns ship their own `<name>.scss` (when needed) using the naming standard:

```scss
.empty-state {
  display: var(--empty-state-display, flex);
  flex-direction: column;
  gap: var(--empty-state-gap, 1rem);
  padding-block: var(--empty-state-padding-block, 3rem);
  text-align: center;
}
```
